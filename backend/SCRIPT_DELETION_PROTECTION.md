# 脚本删除保护机制

## 概述

系统已实现脚本删除保护机制，防止误删除正在被使用的脚本。

## 保护规则

在删除脚本时，系统会检查以下依赖关系：

### 1. 工作流依赖检查

**检查内容**：是否有工作流节点正在使用此脚本

**检查范围**：
- 所有工作流（包括已禁用的工作流）
- 所有节点类型中引用了该脚本的节点

**触发条件**：
```python
WorkflowNode.query.filter_by(script_id=script_id).all()
```

**错误消息示例**：
```
无法删除脚本：以下工作流正在使用此脚本：数据处理流程, 日志分析工作流
```

### 2. 定时任务依赖检查

**检查内容**：是否有启用的定时任务正在使用此脚本

**检查范围**：
- 所有 `enabled=True` 的定时任务

**触发条件**：
```python
Schedule.query.filter_by(script_id=script_id, enabled=True).all()
```

**错误消息示例**：
```
无法删除脚本：以下定时任务正在使用此脚本：每日数据备份, 每小时日志清理
```

## 实现细节

### API 端点
```
DELETE /api/scripts/<script_id>
```

### 检查顺序
1. 检查工作流依赖
2. 检查定时任务依赖
3. 删除脚本工作目录
4. 删除数据库记录

### 返回值

**成功删除**：
```json
{
  "code": 0,
  "message": "脚本删除成功"
}
```

**被工作流使用**：
```json
{
  "code": 1,
  "message": "无法删除脚本：以下工作流正在使用此脚本：工作流1, 工作流2"
}
```
HTTP 状态码: 400

**被定时任务使用**：
```json
{
  "code": 1,
  "message": "无法删除脚本：以下定时任务正在使用此脚本：任务1, 任务2"
}
```
HTTP 状态码: 400

## 如何解除依赖

### 解除工作流依赖

1. 方式一：修改工作流，移除使用该脚本的节点
   - 进入工作流编辑页面
   - 删除或替换使用该脚本的节点
   - 保存工作流

2. 方式二：删除整个工作流
   - 如果工作流不再需要，直接删除工作流

### 解除定时任务依赖

1. 方式一：禁用定时任务
   ```
   POST /api/schedules/<schedule_id>/toggle
   ```
   - 禁用后可以删除脚本

2. 方式二：修改定时任务
   - 更换定时任务使用的脚本

3. 方式三：删除定时任务
   ```
   DELETE /api/schedules/<schedule_id>
   ```

## 数据库约束

### WorkflowNode 表

```python
script_id = db.Column(db.Integer, db.ForeignKey('scripts.id'), nullable=True)
```

- **外键约束**：`scripts.id`
- **可空**：是（因为不是所有节点都需要脚本）
- **级联删除**：未设置（由应用层控制）

### Schedule 表

```python
script_id = db.Column(db.Integer, db.ForeignKey('scripts.id'), nullable=False)
```

- **外键约束**：`scripts.id`
- **可空**：否（定时任务必须关联脚本）
- **级联删除**：未设置（由应用层控制）

## 前端处理

前端会显示后端返回的错误消息，提示用户哪些工作流或定时任务正在使用该脚本。

建议前端实现：
1. 显示依赖列表，让用户点击跳转
2. 提供"强制删除"选项（需要额外确认）
3. 在删除前预先检查并警告用户

## 注意事项

⚠️ **重要**：
1. 即使工作流被禁用，也会阻止脚本删除
2. 只有启用的定时任务会阻止删除（已禁用的不会）
3. 历史执行记录不会阻止删除（执行记录只保存执行时的快照）

## 测试场景

### 场景 1：尝试删除被工作流使用的脚本
```bash
# 1. 创建工作流，使用脚本 ID=1
# 2. 尝试删除脚本 ID=1
curl -X DELETE http://localhost:5000/api/scripts/1

# 预期结果：返回 400，提示工作流名称
```

### 场景 2：尝试删除被定时任务使用的脚本
```bash
# 1. 创建定时任务，使用脚本 ID=2
# 2. 尝试删除脚本 ID=2
curl -X DELETE http://localhost:5000/api/scripts/2

# 预期结果：返回 400，提示定时任务名称
```

### 场景 3：成功删除未被使用的脚本
```bash
# 1. 创建脚本 ID=3，不添加任何依赖
# 2. 删除脚本 ID=3
curl -X DELETE http://localhost:5000/api/scripts/3

# 预期结果：返回 200，删除成功
```
